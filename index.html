ESP32 Weather & Air Quality Monitoring System

An IoT solution showcased at the IoT Exhibition 2025

Project Overview

This project demonstrates a complete IoT weather and air quality monitoring system using ESP32 microcontroller with various sensors. The system collects environmental data in real-time, processes it, and presents it through a modern React.js dashboard, with data storage in a MySQL database accessed via PHP backend.

Hardware Components

ESP32 Microcontroller


ESP32


The ESP32 serves as the brain of the project, featuring:

  • Dual-core processor running at 240MHz
  • Built-in Wi-Fi and Bluetooth capabilities
  • Multiple GPIO pins for connecting sensors
  • Low power consumption for extended battery life

DHT11 Temperature & Humidity Sensor


DHT11 Photo

The DHT11 sensor measures:

  • Temperature range: 0-50°C with ±2°C accuracy
  • Humidity range: 20-90% RH with ±5% accuracy
  • Sampling rate: 1Hz (one reading every second)

MQ-135 Air Quality Sensor


MQ-135 Photo

The MQ-135 detects:

  • Air quality index
  • NH3, NOx, alcohol, benzene, smoke, CO2
  • Analog output for precise readings

Buzzer Alert System


Buzzer Photo

Features:

  • Activates when air quality or temperature exceeds thresholds
  • Adjustable sound frequency
  • Can be muted via the web interface

LED Indicator Lights


LED Lights Photo

Features:

  • Green: Normal conditions Safe
  • Yellow: Warning conditions Warning
  • Red: Critical conditions Alert
  • Blue: System status indicator

Power Supply


Power Supply Photo

Features:

  • 5V DC input via USB
  • Optional battery backup (3.7V LiPo)
  • Low power mode for extended operation

System Architecture

Data Flow

  1. Data Collection: ESP32 reads sensor data (temperature, humidity, air quality) at regular intervals
  2. Local Processing: Raw sensor data is processed on the ESP32 to calculate derived metrics
  3. Data Transmission: ESP32 sends data to the server via HTTP POST requests
  4. Backend Storage: PHP scripts receive data and store it in MySQL database
  5. Frontend Display: React.js application fetches and visualizes data in real-time

Alert System

The system features a dual-alert mechanism:

  • Physical alerts: Buzzer sounds and LEDs change color when thresholds are exceeded
  • Digital alerts: Web dashboard displays warnings and can send email notifications

Backend (PHP & MySQL)

The backend consists of PHP scripts for handling data storage and retrieval:

<?php
// post-data.php - Receives data from ESP32

// Database connection
$servername = "localhost";
$username = "db_user";
$password = "db_password";
$dbname = "weather_monitoring";

// Create connection
$conn = new mysqli($servername, $username, $password, $dbname);

// Check connection
if ($conn->connect_error) {
  die("Connection failed: " . $conn->connect_error);
}

// Get data from request
$data = json_decode(file_get_contents('php://input'), true);

// Insert data into database
$sql = "INSERT INTO sensor_readings (temperature, humidity, air_quality, timestamp) 
        VALUES (?, ?, ?, NOW())";
$stmt = $conn->prepare($sql);
$stmt->bind_param("ddi", $data['temperature'], $data['humidity'], $data['airQuality']);
$stmt->execute();

// Return success/error message
if ($stmt->affected_rows > 0) {
  echo json_encode(["status" => "success"]);
} else {
  echo json_encode(["status" => "error", "message" => $conn->error]);
}

$conn->close();
?>

Database Schema

The MySQL database uses the following schema to store sensor readings:

Column Name Data Type Description
id INT (PRIMARY KEY) Unique identifier for each reading
temperature DECIMAL(5,2) Temperature in degrees Celsius
humidity DECIMAL(5,2) Relative humidity in percentage
air_quality INT Air quality index from MQ-135 sensor
timestamp DATETIME Time when the reading was recorded
alert_status ENUM('normal', 'warning', 'critical') Alert status based on thresholds 
  CREATE TABLE sensor_readings (
      id INT AUTO_INCREMENT PRIMARY KEY,
      temperature DECIMAL(5,2) NOT NULL,
      humidity DECIMAL(5,2) NOT NULL,
      air_quality INT NOT NULL,
      timestamp DATETIME NOT NULL,
      alert_status ENUM('normal', 'warning', 'critical') NOT NULL DEFAULT 'normal'
  );

Setup Instructions

Hardware Assembly

  1. Connect the DHT11 sensor to the ESP32:
    • VCC to 3.3V
    • GND to GND
    • DATA to GPIO4
  2. Connect the MQ-135 sensor:
    • VCC to 5V
    • GND to GND
    • AOUT to GPIO34
  3. Connect the LEDs with appropriate resistors:
    • Green LED to GPIO27
    • Yellow LED to GPIO26
    • Red LED to GPIO25
  4. Connect the buzzer to GPIO12

Software Installation

  1. Install Arduino IDE and ESP32 board package
  2. Install required libraries:
    • DHT sensor library by Adafruit
    • ArduinoJson by Benoit Blanchon
    • WiFi and HTTPClient (included with ESP32 package)
  3. Upload the ESP32 firmware
  4. Set up MySQL database on your server
  5. Deploy PHP backend scripts
  6. Deploy React.js frontend application
